# 用户使用场景路由

> 本文件用于把用户的自然语言问题路由到合适的课程模块。所有回答仅作《伤寒论》课程学习与中医理论整理，不作诊断、处方或个人用药建议。

## 总体规则

1. 先判断用户入口：症状/案例、方剂、六经、逐课复习、概念、截图证据、学习计划。
2. 再打开对应模块，不要一次加载全部资料。
3. 涉及真实身体不适时，必须先声明学习用途，并提醒面诊执业医师。
4. 涉及急下存阴、四逆汤辈、附子、癌症/肿瘤、孕产妇、儿童、危急症时，必须给高风险提示，不输出可直接照做的剂量和处方。
5. 用户要求“出处”“板书”“截图佐证”时，检索 `screenshot-evidence.md`，返回课程、时间点和相对截图路径。

## 场景一：症状辨证/案例学习

用户常见问法：

- “发烧恶寒无汗怎么办？”
- “便秘但小便清白，按课程怎么辨？”
- “女性经期感冒为什么不能发汗？”
- “拉清水是不是寒利？”

处理流程：

1. 打开 `symptom-index.md`，按主诉定位候选症状群。
2. 打开 `six-channel.md`，判断可能处于太阳、少阳、阳明、太阴、少阴、厥阴哪一层。
3. 打开 `formula-patterns.md`，列出课程中对应方证与鉴别方。
4. 若用户需要证据，检索 `screenshot-evidence.md`。
5. 输出结构：
   - 课程辨证方向：可能属于哪个六经/病机。
   - 必问缺失信息：汗、恶寒、口渴、小便、大便、腹诊、脉象、经期等。
   - 方证对照：只讲课程理论，不给个人处方。
   - 鉴别点和禁忌。
   - 截图证据路径，如有需要。

示例答法骨架：

```markdown
这是课程里的辨证学习，不是诊断。按倪师《伤寒论》课程，先看几个分水岭：

| 分水岭 | 偏向 | 课程意义 |
| --- | --- | --- |
| 有汗/无汗 | ... | ... |
| 小便深黄/清白 | ... | ... |
| 腹部拒按/喜按 | ... | ... |

课程上可能对应的方证是 ...，但还缺 ... 才能鉴别。若出现高热神昏、剧痛、脱水、胸痛、呼吸困难等，请立即就医。
```

## 场景二：方剂查询/方证卡片

用户常见问法：

- “麻黄汤和桂枝汤怎么分？”
- “大承气汤什么时候用，什么时候停？”
- “四逆汤辈有哪些风险？”
- “小柴胡汤在课程里怎么讲？”

处理流程：

1. 打开 `formula-patterns.md`，优先找方名。
2. 需要辨证背景时打开 `six-channel.md`。
3. 需要症状入口时打开 `symptom-index.md`。
4. 需要板书证据时检索 `screenshot-evidence.md` 的方名。

回答结构：

- 方剂定位：所属六经/病机。
- 核心方证：主症、关键舌脉/大小便/腹诊。
- 课程方义：药物组合的方向性解释。
- 鉴别方：相似方如何分。
- 禁忌风险：哪些情况课程明确不能用或要特别小心。
- 证据索引：课程 + 时间 + 截图路径。

## 场景三：六经辨证复习

用户常见问法：

- “六经怎么从表到里理解？”
- “太阳、阳明、少阳怎么区分？”
- “太阴少阴厥阴的核心差别？”

处理流程：

1. 打开 `six-channel.md`。
2. 需要方剂时再打开 `formula-patterns.md`。
3. 需要按课次学习时打开 `lesson-map.md`。

输出重点：

- 先给总表：病位、主症、病机、代表方、禁忌。
- 再解释传变：太阳表证、少阳半表半里、阳明里实热、三阴里虚寒/寒热错杂。
- 最后给复习路线：对应课程课次。

## 场景四：逐课复习/学习计划

用户常见问法：

- “第 07 讲讲了什么？”
- “帮我从 07 开始复习。”
- “给我按一周做伤寒论复习计划。”

处理流程：

1. 打开 `lesson-map.md`。
2. 按课次输出“主题、重点方证、必背分水岭、回看截图关键词”。
3. 若用户要深入某课，检索该课在 `screenshot-evidence.md` 的条目。

## 场景五：截图证据/板书索引

用户常见问法：

- “这个有截图吗？”
- “把讲小柴胡汤加减的板书找出来。”
- “给我看热结旁流的图。”

处理流程：

1. 优先运行 `python scripts/search_screenshots.py <自然语言问题或关键词...>` 做跨截图索引排序；需要确认拆词时加 `--show-terms`。
2. 优先返回 3 到 8 条最相关证据，格式固定：

```markdown
- 伤寒论05 82:42：小柴胡汤加减法板书
  截图路径：assets/screenshots/shanghan/...
```

3. 如果当前环境支持图片打开，且用户要求“显示”，打开对应截图；否则提供公开仓库内的相对路径。
4. 截图只作为课程笔记索引，不当作医学判断依据。

## 场景六：课程蒸馏/知识库扩展

用户常见问法：

- “把这部分整理成卡片。”
- “继续蒸馏成更细模块。”
- “做成我以后能问的 skill。”

处理流程：

1. 使用 `lesson-map.md` 确认课次范围。
2. 使用 `symptom-index.md` 和 `formula-patterns.md` 抽出可检索卡片。
3. 使用 `screenshot-evidence.md` 绑定证据路径。
4. 新增或更新 references 文件时，保持 `SKILL.md` 只放路由和安全规则，详细知识放 references。
